# 项目上下文

## 1. 项目定位

当前项目已经不再是一个单纯的 RAG Demo，而是逐步演进成一个面向“知识库问答 + 题库生成 + 面试训练 + 训练分析”的综合型练习系统。

它的核心价值在于：

1. 支持用户围绕自己的资料构建私有知识库
2. 基于知识库进行 RAG 问答
3. 在问答之外延展到面试训练场景

## 2. 当前已经具备的 RAG 主链路

### 2.1 知识库与入库

- 支持 `txt / md / pdf` 上传
- 支持文本清洗与 section-based chunk 构建
- 支持 embedding 生成与 PGVector 落库
- 支持文档删除时同步删除 chunk

### 2.2 检索与回答

- 支持基于知识库的问答
- 支持引用片段返回
- 支持问题意图路由
- 支持 rerank
- 支持在检索不足时进行受控补充回答

### 2.3 可解释性与调试

- 支持 chunk 查看
- 支持 debug trace
- 支持 retrieval steps / reranked results / structured context 返回
- 支持围绕检索质量的回归测试

## 3. 当前项目的讲述重点

如果把项目用于面试展示，最值得强调的不是“功能很多”，而是：

- RAG 主链路已经跑通
- 已经做过真实的检索质量优化
- 已经解决过具体的工程问题，例如：
  - Markdown 前言内容丢失
  - 定义型 chunk 未排到前列
  - 引用片段被粗暴截断
  - 向量存储迁移到 PGVector

## 4. 当前最值得继续投入的方向

- chunk 语义切分
- metadata 增强
- hybrid retrieval
- 更强 rerank
- 更系统的离线评估

如果继续把项目打磨成更适合面试展示的作品，建议继续把重心放在：

- RAG 质量优化
- RAG 可解释性
- RAG 工程化能力
